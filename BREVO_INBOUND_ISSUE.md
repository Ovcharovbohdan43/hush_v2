# ⚠️ Проблема: Brevo может не поддерживать входящие письма

## Текущая ситуация

В настройках домена `hush.autos` в Brevo есть только DNS записи для **отправки** писем (DKIM, DMARC), но нет настроек для **получения** входящих писем.

## Возможные решения

### Вариант 1: Проверить, есть ли Inbound Email в Brevo

1. В панели Brevo проверьте:
   - Settings → Inbound Email
   - Settings → Email → Inbound
   - Или в разделе Domains → ваш домен → Inbound settings

2. Если есть опция "Inbound Email" или "Receive emails":
   - Настройте MX записи для получения писем
   - Настройте webhook для входящих писем

### Вариант 2: Использовать другой email провайдер для входящих писем

Если Brevo не поддерживает входящие письма, можно использовать:

#### Mailgun (рекомендуется)
- ✅ Хорошая поддержка входящих писем
- ✅ Простая настройка webhook
- ✅ Бесплатный тариф: 5000 писем/месяц

#### SendGrid
- ✅ Поддержка входящих писем
- ✅ Webhook для входящих писем
- ✅ Бесплатный тариф: 100 писем/день

#### Cloudflare Email Routing (бесплатно)
- ✅ Полностью бесплатно
- ✅ Простая настройка
- ✅ Пересылка на любой email

### Вариант 3: Использовать Brevo только для отправки, другой сервис для получения

1. **Brevo** - для отправки писем (уже настроено)
2. **Mailgun/SendGrid** - для получения входящих писем на домен `hush.autos`

## Рекомендуемое решение: Mailgun

### Шаги настройки Mailgun:

1. **Зарегистрируйтесь в Mailgun:**
   - https://www.mailgun.com/
   - Бесплатный тариф: 5000 писем/месяц

2. **Добавьте домен `hush.autos`:**
   - Mailgun → Sending → Domains → Add New Domain
   - Добавьте `hush.autos`

3. **Настройте DNS записи:**
   - Mailgun покажет MX записи для получения писем
   - Добавьте их в DNS вашего домена
   - Также добавьте TXT записи для верификации

4. **Настройте webhook:**
   - Mailgun → Sending → Webhooks
   - Добавьте webhook: `https://your-ngrok-url.ngrok.io/api/v1/incoming/mailgun`
   - Событие: "Inbound messages"

5. **Обновите код сервера:**
   - Убедитесь, что endpoint `/api/v1/incoming/mailgun` работает
   - Уже реализован в `server/src/api/incoming.rs`

## Быстрая проверка Brevo

Попробуйте найти в панели Brevo:
- Settings → Email → Inbound
- Settings → Inbound Email
- Domains → hush.autos → Inbound settings
- Или обратитесь в поддержку Brevo

## Что делать сейчас

1. **Проверьте панель Brevo** на наличие раздела Inbound Email
2. **Если нет** - настройте Mailgun для получения писем
3. **Brevo оставьте** для отправки писем (уже настроено)

## Альтернатива: Cloudflare Email Routing (самый простой)

Если хотите быстрое решение:

1. Используйте Cloudflare Email Routing (бесплатно)
2. Настройте пересылку всех писем на `*@hush.autos` на ваш сервер
3. Используйте API Cloudflare для получения писем
